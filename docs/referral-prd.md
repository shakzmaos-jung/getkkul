# PRD — getkkul 친구추천 크레딧 프로그램

**버전** v0.1 · **작성자** Chess · **상태** v1 스코프 확정 · **기반** getkkul PRD v0.4
**코드네임** 친구추천(referral) + 크레딧(credit)

---

## 1. 한 줄 요약

사용자가 자기 추천 링크로 친구를 데려오고, 친구가 활성화 조건을 충족하면 **추천인·피추천인 모두에게 비현금 크레딧 2,000원**이 지급되는 양방향 추천 프로그램. 크레딧은 향후 유료 결제 시 할인에 사용된다. 목적은 겟꿀의 빠른 사용자 확산과, 그 과정에서 쌓이는 활성 사용자 지표 확보다.

## 2. 배경 & 목적

겟꿀을 빠르게 바이럴시켜 사용자·트래픽을 확보하고, 그 지표를 커리어 포트폴리오로 활용한다. 핵심은 단순 가입이 아니라 **실제로 서비스를 쓰는 활성 사용자**를 늘리는 것이다. 따라서 보상은 "가입"이 아니라 "활성화(채널 3개 구독 + 요약 10개 수신)" 시점에 지급해, 어뷰징을 막고 지표 품질을 높인다. 비용은 현금이 아니라 유료 결제 할인용 크레딧이므로, 실현 부담은 명목보다 훨씬 작다(대다수 미전환·미사용, breakage).

## 3. 목표 (Goals)

- 사용자가 자기 추천 링크를 생성·공유하고, 친구가 그 링크로 가입·활성화하면 양쪽에 크레딧을 지급한다.
- 보상을 활성화 시점에 지급해 진짜 활성 사용자에게만 비용이 발생하게 한다.
- 총 발행 예산(500만원)을 코드로 강제해 비용을 확정적으로 묶는다.
- 사용자가 자기 크레딧 내역·잔액·만료와, 추천한 친구들의 진행 현황을 설정 화면에서 확인한다.

## 4. 비목표 (Non-goals) — v1에서 하지 않는 것

- 다단계(2단계 이상) 보상 — 단일 단계 양방향만.
- 크레딧의 현금 환급·양도.
- 실제 유료 결제 연동 — 유료 티어가 아직 없다. 크레딧 적립·원장·차감 로직은 구현하되, 결제 시 실제 차감은 유료화 시점에 연결(v2). 
- 추천인에게 피추천인의 상세 활동(어떤 채널을 구독했는지 등) 노출.

## 5. 핵심 플로우

1. 사용자가 "공유하기"를 누르면 본인 고유 난수코드가 박힌 추천 링크가 생성된다.
2. 친구가 그 링크로 접속해 회원가입(Google/이메일)한다 → 추천 관계가 pending으로 기록된다.
3. 친구가 활성화 조건(채널 구독 3개 + 요약 항목 10개 수신)을 충족한다.
4. 활성화 시 추천인·피추천인 각각에게 크레딧 2,000원이 지급된다(가드 통과 시).
5. 지급 시 수령자에게 이메일/푸시 알림이 발송된다.
6. 크레딧·추천현황은 설정 메뉴에서 확인한다. 크레딧은 향후 유료 결제 할인에 사용된다는 안내를 표기한다.

## 6. 보상 정책 (확정)

| 항목 | 내용 |
|---|---|
| 추천인 / 피추천인 보상 | 각 2,000원 (활성화 시) |
| 활성화 조건 | 채널 구독 3개 + 요약 항목 10개 수신 |
| 추천 인원 | 무제한 (1인 상한까지) |
| 1인당 획득 상한 | 50,000원 (최대 25명 추천분) |
| 전체 예산 킬스위치 | 500만원 (누적 발행 도달 시 신규 지급 중단) |
| 결제 사용 상한 | 결제액의 50%까지 크레딧 차감 |
| 유효기간 | 지급 건별 5년, 만료 임박 순 차감(FIFO) |
| 월 구독가(가정) | 2,000원 |

## 7. 비용 & 예산 모델

활성 추천 가입자 1명당 명목 비용 = 4,000원(본인 2,000 + 추천인 2,000). 오가닉(추천 없는) 가입은 비용 0. 500만원 예산은 활성 추천 가입자 약 1,250명까지 커버한다(1,000명 = 400만원). 실현 비용은 명목보다 훨씬 작다: 크레딧은 유료 전환 + 실제 사용 시에만, 그것도 결제액 50%까지만 소진되며, 월 2,000원·연 최대 12,000원 소진 속도상 50,000원 상한을 다 쓰려면 4년 넘게 유료 유지해야 하므로 대다수가 상한을 소진하지 못한다(breakage).

## 8. 리스크 & 안전장치

- **예산 킬스위치**: 누적 발행 500만원 도달 시 신규 추천 크레딧 지급 자동 중단.
- **1인당 상한 50,000원**: 한 명이 예산을 독식하는 집중 리스크 차단.
- **결제 사용 상한 50%**: 회당 실현 손실을 절반으로 묶고, 사용자가 항상 일부는 현금 결제.
- **활성화 게이트**: 가짜 가입으로는 조건 미충족 → 비용 미발생.
- **재가입 어뷰징 방지**: 정규화 이메일 해시 + "보상 이력" 플래그를 탈퇴 후에도 보존, 기기·결제수단 지문. 같은 사람 재가입 시 재보상 차단.
- **탈퇴 시 크레딧 소멸**: 본인 보유 크레딧 즉시 소멸(그가 유발한 추천인 보상은 유지).
- **유효기간 5년**: 미상환 부채 꼬리 제한.

## 9. 규제 & 프라이버시

- 구조는 비현금 크레딧 + 구매요건 없음 + 단일 단계 양방향이라, 현금·다단계 판매(방문판매법) 영역과 거리가 있다. 단계는 2단 이하·비현금을 유지한다.
- 어뷰징 방지를 위해 탈퇴 후에도 정규화 이메일 해시 등 최소 식별 정보를 보존한다. 이는 개인정보 최소수집 원칙의 정당한 예외이며, 개인정보처리방침에 "부정 이용 방지 목적의 최소 식별 해시 보관" 근거 문구를 추가한다.

## 10. 성공 지표

- 추천으로 유입되어 **활성화까지 도달한 사용자 수**(핵심 포트폴리오 지표).
- 추천 링크 공유→가입→활성화 전환율(퍼널).
- 활성 추천 가입자 1명당 실현 크레딧(명목 대비 breakage율).

## 11. v1 확정 사항

보상 각 2,000원 / 활성화=채널3+요약10 / 1인 상한 50,000원 / 예산 500만 킬스위치 / 결제 50% 사용 / 유효 5년 FIFO / 크레딧·추천현황 조회(진행률만) / 재가입 해시 보존 / 탈퇴 시 본인 크레딧 소멸. 실제 결제 차감 연동은 유료화(v2)로 이연.
