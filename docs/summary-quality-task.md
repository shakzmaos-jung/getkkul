# 작업 지시 — getkkul 다이제스트 요약 품질 개선

**대상 에이전트** Claude Code · **상태** 실행 준비 완료

---

## 목표

다이제스트 요약을 "문장 수 기반"에서 "정보 계층 기반"으로 재설계한다. 길이 역전 제거, long의 사실↔인사이트 단락 구분, 적응형 깊이, 보수적 용어 교정, 핵심 문장 하이라이트, 그리고 👍/👎 피드백 루프로 시스템 프롬프트를 지속 개선한다. 다이제스트는 getkkul의 핵심 지불가치다.

## 단일 진실 공급원

- `docs/summary-quality-spec.md` — 이 개선의 REQ/AC와 검증 기준(S1~S7). 모든 결정이 여기 있다.
- `CLAUDE.md`, `docs/PRD.md`, `docs/SSR.md` — 전반 규칙·스택·격리 경계.
- 현재 코드: `lib/summary/summarize.ts`, `format.ts`, summaries/videos/content_terms 스키마.

## 작업 방식

- 스키마 변경(content_feedback 등)·프롬프트 재작성을 포함하지만 결정은 스펙에 확정돼 있다. 스펙 범위 안에서 추가 승인 없이 끝까지 자율 완성한다(TDD 준수).
- 시작 전, 내가 정하거나 해줄 것이 있으면 하나로 모아 물어보고(없으면 없다고 말하고 진행), 그 외엔 스펙 밖 에스컬레이션 사유가 생길 때만 멈춘다.

## 확정된 결정 (스펙 반영 — 재확인 불필요)

- 정보 계층: short=TL;DR / normal=핵심+맥락 / long=[핵심 사실 요약] + [맥락·시사점·인사이트] **2단락 구분**.
- 단조성(short ≤ normal ≤ long) 보장. 문장 수 범위 검증을 단조성 검증으로 교체·보강. 역전은 사용자에게 노출 금지.
- 적응형 깊이: 빈약 콘텐츠는 long 무리 생성 금지 + "짧아서 짧게만 제공" 안내. 상위 모드 "제공 안 함" 상태 저장.
- 보수적 용어 교정: 채널 도메인 힌트로 명백한 오류만 교정. **과교정 금지**(없던 개체 삽입 금지, 불확실 시 원문 유지). content_terms 활용 가능.
- long 핵심 문장 하이라이트(스키마 마킹 → underline 렌더).
- 피드백 루프: 콘텐츠 카드 하단 👍/👎, (user_id, video_id, length_mode, language)와 함께 저장, 집계 지표로 프롬프트 개선(HOTL: 지표→사람이 개정→회귀 테스트).
- long 시스템 프롬프트 재작성(정보 계층·단락·근거·교정·하이라이트 지시) + **프롬프트 개선안을 반드시 산출물에 포함**.
- 확장: 용어 설명·타임스탬프·on-demand "더 자세히/더 쉽게"(AI질의와 역할 구분). on-demand는 결과 캐시.
- long에 한해 reasoning_effort='low'(그 외 minimal 유지). Batch API 미사용(실시간 우선). 오디오 STT 유료 API 유지(비용 관측).

## 산출물에 반드시 포함 (Chess 요청)

- **개선된 long 시스템 프롬프트 문안**(실제 코드에 반영 + 문서에 명시). 정보 계층·2단락·근거 준수·보수적 교정·하이라이트를 지시하는 형태.
- 프롬프트 버전 관리 및 회귀 테스트.

## 품질 기준 (TDD)

- 각 AC를 실패 테스트로 먼저 작성한 뒤 구현한다.
- **반드시 테스트 우선(무결점)**:
  - "단조성: short ≤ normal ≤ long, 역전 0건 + 위반 시 방어".
  - "보수적 교정: 명백 오류 교정 + 불확실·유사발음 개체 삽입 0건(과교정 방지)".
- 그 외 테스트 우선: long 2단락 구분 저장, 적응형 depth 판정·상위모드 제공안함, 하이라이트 마킹, 피드백 저장·변경·집계.
- 요약은 단일 호출 유지(전사 1회 전송, 3종 생성). 재시도는 전사 미포함.
- 시크릿은 env로만.

## 수용성 검증

- spec J절 S1~S7을 검증 대상으로. 통과 기준: S1(역전 0)·S4(과교정 0) 무결점, 나머지 전부 통과.
- 자동 계약 테스트(단조성·교정·단락·적응형·하이라이트·피드백 저장)와, 사람이 실제 몇 개 영상으로 품질을 눈으로 확인하는 체크리스트(long 단락 구분·인사이트 품질·용어 교정 사례·👍👎 동작)를 함께 제시한다.

## 문서 정합성

- 요약 구조가 바뀌므로 `docs/SSR.md`의 요약 관련 AC(길이/형식)를 정보-계층·단조성·2단락에 맞춰 갱신한다. 변경은 `docs/adr/`에 기록.
