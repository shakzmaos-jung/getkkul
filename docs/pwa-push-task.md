# 작업 지시 — getkkul PWA 설치 + 모바일 푸시알림

**대상 에이전트** Claude Code · **상태** 실행 준비 완료

---

## 목표

getkkul 웹 서비스에 (1) PWA 설치 기능과 (2) 모바일 푸시알림(07:30 / 11:30 / 17:30 슬롯별 멀티체크)을 추가한다. iOS·Android 모두에서 동작해야 한다.

## 단일 진실 공급원

- `docs/pwa-push-spec.md` — 이 기능의 REQ/AC(수용 기준). 모든 결정이 여기 확정돼 있다.
- `CLAUDE.md`, `docs/PRD.md`, `docs/SSR.md` — 프로젝트 전반 규칙·스택·격리 경계.
- 위 문서의 규칙·스택·격리 경계를 따른다.

## 작업 방식 (중요)

- 이 작업은 스키마 변경·외부 의존성 추가·시크릿을 포함하지만, **관련 결정이 스펙에 이미 승인돼 있다.** 따라서 스펙 범위 안에서는 추가 승인 없이 끝까지 자율로 완성한다(TDD 준수).
- 중간에 매 단계 확인을 요청하지 않는다. 스펙에 명시되지 않은 새로운 에스컬레이션 사유(스펙 밖 스키마 변경, 스펙에 없는 새 의존성 등)가 생길 때만 멈추고 묻는다.

## 유일한 사전작업 게이트 (VAPID)

시작 시 딱 한 가지만 먼저 처리한다.

1. `npx web-push generate-vapid-keys`로 VAPID 키쌍을 생성해 값을 출력한다.
2. 나(사용자)가 그 값을 로컬 `.env`, GitHub Actions secrets, Vercel 환경변수 세 곳에 넣겠다.
   - `VAPID_PUBLIC_KEY`, `VAPID_PRIVATE_KEY`, `VAPID_SUBJECT`(= `mailto:shakzmaos@gmail.com`)
3. 이 키가 등록되기 전에는 **실제 푸시 발송 연동 테스트만** 보류한다. 그 외 구현(매니페스트·서비스워커·아이콘 파생·설치 버튼·OS 다이얼로그·구독·설정 토글·notify 푸시 채널·발송 연동 코드·단위 테스트)은 즉시 진행한다.

## 확정된 결정 (스펙에 반영됨 — 재확인 불필요)

- 앱 아이콘: 기존 getkkul 아이콘 사용(192/512/maskable 파생 생성).
- 프로덕션 URL: https://getkkul.vercel.app (HTTPS).
- 스키마: `push_subscriptions` 테이블 신설 + `user_settings`에 `push_slot_0730/1130/1730`(기본 false), `skip_empty_push`·`skip_empty_email`(기본 true) 추가.
- 의존성: `web-push` 추가.
- notify 구조: 기존 이메일 `Notifier` 유지 + 푸시용 인터페이스 병렬 추가 + 발송 오케스트레이터가 채널 선택.
- 푸시 vs 이메일: 독립 운영(중복 허용).
- 빈 슬롯 처리: 설정 토글로 제어(`skip_empty_push` / `skip_empty_email`, 기본 생략).
- 설치 버튼 UX: 상단 메뉴에 다운로드 아이콘 + "앱 설치" 텍스트. 클릭 시 OS 선택 다이얼로그 → Android는 실제 설치 프롬프트, iOS는 "공유 → 홈 화면에 추가" 수동 가이드(자동 다운로드 불가).

## 구현 범위

`docs/pwa-push-spec.md`의 REQ A~H를 모두 구현한다. 요약하면: PWA 설치 기반(매니페스트·서비스워커·아이콘), 상단 앱 설치 버튼·OS 선택 다이얼로그, 푸시 구독(iOS는 standalone에서만), 슬롯별 발송 설정(멀티체크) + 빈 슬롯 생략 토글, 기존 deliver 플로우에 푸시 채널 통합, lib/notify에 web-push 구현체 추가.

## 품질 기준 (TDD)

- 각 AC를 **실패 테스트로 먼저 작성한 뒤** 구현한다.
- 특히 다음은 반드시 테스트 우선: "슬롯별 푸시 발송 대상 필터링", "iOS standalone 감지 분기", "구독 저장/삭제", "빈 슬롯 생략 설정(skip_empty_push/skip_empty_email) 분기".
- 시크릿은 절대 코드에 넣지 말고 환경변수로만 참조한다.
- 개별 사용자 푸시 실패가 전체 발송 잡을 중단시키지 않는다(격리·회복력).
