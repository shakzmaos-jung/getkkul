# SSR 부록 — getkkul 사이드 메뉴(우측 패널) 재설계

**버전** v0.1 · **작성자** Chess (스펙 초안: 협업) · **상태** v1 스펙 확정(목업 검증 완료) · **기반** getkkul SSR v0.2, 현재 `components/layout/SideMenu.tsx`
**용도** 각 AC는 TDD 테스트의 원천이다. 판정 가능하게 기술한다.
**표기** REQ = 요구사항, AC = 수용 기준.

---

## 0. 배경 & 확정 결정

현재 우측 사이드 패널(`SideMenu.tsx`)은 조악하게 느껴진다. 코드 분석상 원인은 (1) 한 리스트에 상호작용 3종 혼재(서비스소개·개발자정보=아코디언, 설정·라이선스=페이지 이동, 테마=인라인 토글), (2) 사용자 프로필·계정·로그아웃 부재, (3) 앱 버전 정보 부재, (4) 모든 행이 동일 스타일이라 위계·그룹 없음.

**확정 재설계(인터랙티브 목업으로 검증됨)**: 위→아래로 프로필 카드 → "메뉴" 그룹(설정·서비스소개·개발자정보·오픈소스라이선스, 전부 페이지 이동으로 통일·아코디언 폐지) → "화면" 그룹(다크모드 인라인 토글) → "계정" 그룹(로그아웃) → 메타 푸터(버전 + 카피라이트). 섹션 라벨·구분으로 위계를 준다. 기존 접근성·애니메이션은 보존한다.

## A. 패널 구조

**REQ-A1** 패널은 다음 순서·그룹으로 구성된다.
- AC-A1.1 위→아래: ① 프로필 카드 → ② "메뉴" 그룹 → ③ "화면" 그룹 → ④ "계정" 그룹 → ⑤ 메타 푸터.
- AC-A1.2 각 그룹은 섹션 라벨과 시각적 구분(여백/구분)으로 나뉘어 위계가 드러난다(모든 행 동일 스타일 금지).

## B. 상호작용 일관성 (핵심)

**REQ-B1** 상호작용을 유형별로 통일한다.
- AC-B1.1 정보/설정 열람(설정·서비스소개·개발자정보·오픈소스라이선스)은 **모두 동일한 페이지 이동 패턴**을 사용한다(우측 chevron → 표시). 제자리 아코디언은 폐지한다.
- AC-B1.2 즉시 제어(다크모드)는 인라인 토글로 "화면" 그룹에만 둔다.
- AC-B1.3 서비스 소개·개발자 정보는 아코디언이 아니라, 설정·라이선스와 같은 이동 대상(별도 화면/라우트)으로 만든다. (설정=/settings, 라이선스=/licenses 기존 유지. 서비스소개·개발자정보도 동일 방식의 화면으로 — 신규 라우트 또는 기존 컨벤션에 맞는 화면.)

## C. 프로필 카드

**REQ-C1** 패널 최상단에 사용자 프로필 카드를 둔다.
- AC-C1.1 아바타(이니셜 또는 프로필 이미지) + 이름 + 이메일을 표시한다.
- AC-C1.2 현재 멤버십 플랜 배지를 표시한다(예: "얼리버드 무료 · Medium").
- AC-C1.3 카드를 누르면 계정 화면(/account)으로 이동한다.
- AC-C1.4 멤버십 데이터가 아직 없으면 배지를 우아하게 처리한다(기본 "무료" 표시 또는 배지 숨김). 로그인 사용자 정보(이름·이메일)는 세션/프로필에서 가져온다.

## D. "메뉴" 그룹

**REQ-D1** 네 개 항목을 일관된 이동 패턴으로 제공한다.
- AC-D1.1 설정 → /settings (기존).
- AC-D1.2 서비스 소개 → 서비스 소개 화면(아코디언 아님).
- AC-D1.3 개발자 정보 → 개발자 정보 화면(아코디언 아님). 기존 `RememberLogo` TODO 플레이스홀더는 실제 로고나 깔끔한 대체로 교체한다.
- AC-D1.4 오픈소스 라이선스 → /licenses (기존).
- AC-D1.5 네 항목 모두 아이콘 + 라벨 + 우측 chevron(→)으로 시각적으로 동일하게 보인다.

## E. "화면" 그룹

**REQ-E1** 다크모드 토글을 인라인 제어로 둔다.
- AC-E1.1 "화면" 섹션에 다크/라이트 토글이 있고, 그 자리에서 즉시 테마가 바뀐다(기존 ThemeToggle 로직 재사용).

## F. "계정" 그룹

**REQ-F1** 계정 액션을 제공한다.
- AC-F1.1 "계정" 섹션에 로그아웃 항목이 있고, 누르면 기존 로그아웃(세션 종료) 로직을 실행한다.
- AC-F1.2 계정 화면(/account)에도 로그아웃 진입이 있어도 무방(중복 허용).

## G. 메타 푸터

**REQ-G1** 패널 하단에 메타 정보를 둔다.
- AC-G1.1 앱 버전(예: "getkkul v1.0.0")을 표시한다. 버전은 package.json version을 빌드 시 주입(예: NEXT_PUBLIC_APP_VERSION 또는 생성 상수)해 하드코딩하지 않는다.
- AC-G1.2 "© 2026 getkkul · Made in Seoul" 카피라이트를 표시한다.

## H. 보존 요구사항 (회귀 금지)

**REQ-H1** 기존의 잘 된 부분을 유지한다.
- AC-H1.1 포커스 트랩(패널 내 Tab 순환), ESC 닫기, 오버레이 탭 닫기, 우측 스와이프 닫기를 유지한다.
- AC-H1.2 슬라이드 애니메이션은 transform(translate-x) 기반 약 200ms, 블러 없이 가볍게 유지한다.
- AC-H1.3 터치 타깃 최소 44px, 라이트/다크 테마 모두 대응.
- AC-H1.4 하위 화면(서비스소개·개발자정보 등) 진입/복귀도 동일한 접근성·닫기 규칙을 따른다.

## I. 데이터 & 의존성

- 사용자 정보(이름·이메일·아바타): 기존 세션/프로필에서.
- 플랜 배지: 멤버십(subscriptions_membership)에서 현재 플랜. 미구축 시 우아한 폴백(AC-C1.4).
- 버전: package.json version 주입.
- 계정/로그아웃: 기존 /account·인증 로직 재사용.

## J. 검증 기준 (수용성)

- **V1 상호작용 일관성**: "메뉴" 4개 항목이 모두 동일한 이동 패턴으로 동작한다. 패널에 제자리 아코디언이 하나도 남아 있지 않다.
- **V2 프로필**: 프로필 카드가 이름·이메일·플랜 배지를 표시하고, 탭 시 /account로 이동한다. 멤버십 미구축 시 폴백이 동작한다.
- **V3 버전**: 푸터에 package.json 기반 앱 버전이 표시된다(하드코딩 아님).
- **V4 로그아웃**: "계정" 그룹 로그아웃이 실제 세션 종료를 수행한다.
- **V5 위계**: 그룹별 섹션 라벨·구분이 있고, 정보/이동/제어 행이 시각적으로 구분된다.
- **V6 보존**: 포커스 트랩·ESC·오버레이·스와이프 닫기·200ms 슬라이드·44px·라이트/다크가 모두 유지된다(회귀 없음).
- **V7 무회귀**: 기존 /settings·/licenses·/account 라우트가 정상 동작한다.
- 통과 기준: V1~V7 전부 통과. 특히 V1(아코디언 0개)·V6(접근성 보존)은 필수.
