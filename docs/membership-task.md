# 작업 지시 — getkkul 멤버십 구독 & 크레딧 결제

**대상 에이전트** Claude Code · **상태** 실행 준비 완료

---

## 목표

크레딧을 서비스 내 통화로 하는 월 구독 멤버십 시스템(Free/Small/Medium/Large)을 구현한다. 결제일·비례정산·한도집행·해지·PoC 무료 Medium·양채널 알림·조회 화면까지. PG 연동은 향후(지금은 크레딧 결제).

## 단일 진실 공급원

- `docs/membership-spec.md` — 이 기능의 REQ/AC. 모든 결정이 여기 확정.
- `docs/referral-spec.md` — 크레딧 원장(credit_ledger). 구독 결제는 반드시 여기서 FIFO·50% 상한으로 차감.
- `CLAUDE.md`, `docs/PRD.md`, `docs/SSR.md` — 전반 규칙·스택·격리 경계.
- 관련 코드: notify 레이어(결제 알림), 기존 subscriptions(active 상태 추가), 설정·구독 화면.

## 작업 방식 (중요)

- 스키마 변경·크레딧 원장 연동을 포함하지만 결정은 스펙에 승인돼 있다. 스펙 범위 안에서 추가 승인 없이 끝까지 자율 완성한다(TDD 준수).
- 시작 전, 내가 정하거나 해줄 것이 있으면 하나로 모아 물어보고(없으면 없다고 말하고 진행), 그 외엔 스펙 밖 에스컬레이션 사유가 생길 때만 멈춘다.
- **전제**: 이 작업은 referral-spec의 credit_ledger가 먼저 존재해야 정합적으로 붙는다. credit_ledger가 아직 없으면, 먼저 그 의존성을 알리고 순서를 조정한다.

## 확정된 결정 (스펙 반영 — 재확인 불필요)

membership-spec 0절·K절의 모든 정책. 핵심: anchor day 결제일(1/31→2/28·29→3/31, 원래 anchor 복귀), KST 종료 경계(종료일 다음날 00:00), 업그레이드 즉시 비례정산·다운그레이드 다음주기·환급없음, 초과채널 비활성 보관(삭제 금지), 주기 시작일 한도 리셋, 종료 후 Free 강등, 크레딧 부족 유예 3일→Free, 멱등키 주기당 1회, 크레딧 결제=credit_ledger FIFO·50% 상한, PoC 무료 Medium 2026-09-30 23:59:59 종료, 결제 성공·실패 푸시+이메일, "AI 기능"=콘텐츠 카드 우측 상단 AI 질의.

## UX/UI 반영 필수 사항

스펙의 정책을 화면에서 명확히 드러낸다.

- **플랜 관리 화면(별도 메뉴)**: 4개 플랜 비교 카드(요금·채널·다이제스트·AI 질의 한도), 현재 플랜 강조, 다음 결제일·요금 표시.
- **업그레이드 플로우**: 즉시 비례정산 금액(남은 일수 차액)을 **결제 전에 미리 표시**하고 확인받는다. "지금 올리면 남은 N일치 차액 X크레딧 즉시 차감, 즉시 적용".
- **다운그레이드/해지 플로우**: "다음 주기부터 적용, 이번 주기 유지, 환급 없음"을 명확히 고지. 예약 상태 배지 + 예약 취소 버튼.
- **다운그레이드 초과 채널 UX**: 삭제하지 않고 "비활성" 상태로 표시. 활성 유지할 채널을 사용자가 고를 수 있는 UI(자동 선택 + 변경 가능). "업그레이드하면 다시 켜져요" 안내로 재업그레이드 유인.
- **사용량 표시**: 다이제스트 x/한도, AI 질의 y/한도, 채널 z/한도를 진행바로. 한도 임박·초과 시 안내와 업그레이드 CTA.
- **AI 질의 버튼**: 콘텐츠 카드 우측 상단 AI 질의 기능에 남은 횟수/한도 노출. 한도 초과 시 비활성 + 안내.
- **구독·결제 내역(카드형)**: 현재 플랜 카드 + 결제 내역 리스트(일자·플랜·금액·크레딧 차감·상태) + 구독 변경 이력. credit_ledger와 정합.
- **크레딧 잔액·만료 연동**: 결제 화면에서 사용 가능 크레딧·곧 만료 예정 표시(referral-spec의 원장과 연결).
- **PoC 무료 배너**: 무료 Medium 사용자에게 "2026-09-30까지 무료" 표시, 종료 7일 전 안내.
- **결제 실패/유예 상태**: 유예 중임을 명확히 표시(남은 유예일 + 충전 CTA).

## 품질 기준 (TDD)

- 각 AC를 실패 테스트로 먼저 작성한 뒤 구현한다.
- **반드시 테스트 우선(무결점 요구)**:
  - "anchor day 결제일 계산" — 시작일 28/29/30/31 × 2월·윤년·30일달·31일달 전 조합. off-by-one·타임존 오류 0건.
  - "업그레이드 비례정산 금액 계산" — 남은 일수 차액 정확.
  - "결제 멱등성" — 잡 중복·버튼 중복에도 1회 차감.
  - "크레딧 원장 정합" — 구독 결제가 credit_ledger에 FIFO·50% 상한 usage로 정확 반영, 이중장부 없음.
  - "다운그레이드 초과 채널 비활성 보관 + 재업그레이드 복구".
  - "크레딧 부족 유예 3일 → Free 강등".
  - "PoC 무료 종료 경계(2026-09-30 23:59:59 KST) 전환".
- 날짜·시각은 전부 KST 기준으로 계산·저장(UTC 저장 시 변환 명시). "24시" 같은 모호 시각 금지.
- 시크릿은 env로만. 결제 관련 쓰기는 트랜잭션으로 원자적.

## 수용성 검증

- membership-spec J절 S1~S11을 검증 대상으로 삼는다. 통과 기준: 전부 통과, 특히 날짜(S1·S2)·멱등(S7)·크레딧 정합(S11) 무결점.
- 자동화 계약 테스트(날짜·정산·멱등·원장은 전부 자동 테스트)와, 사람이 확인할 플로우 체크리스트(업/다운/해지/유예/PoC종료 화면 동작)를 함께 제시한다.

## 문서 정합성

- 결제/구독이 credit_ledger에 의존하므로, referral-spec과의 연결(usage 트랜잭션·50% 상한)을 코드·문서에서 일치시킨다. 변경은 `docs/adr/`에 기록.
