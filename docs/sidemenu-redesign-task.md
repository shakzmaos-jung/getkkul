# 작업 지시 — getkkul 사이드 메뉴(우측 패널) 재설계

**대상 에이전트** Claude Code · **상태** 실행 준비 완료

---

## 목표

우측 사이드 패널(`components/layout/SideMenu.tsx`)의 조악한 UX를 개선한다. 상호작용을 유형별로 통일(정보=이동, 제어=인라인, 정체성=상단 카드)하고, 프로필·버전·계정 정보를 추가하며, 그룹·위계를 준다. 기능 재작성이 아니라 정보구조·시각·일관성 개선이다.

## 단일 진실 공급원

- `docs/sidemenu-redesign-spec.md` — 이 재설계의 REQ/AC와 검증 기준(V1~V7). 모든 결정이 여기 있다.
- `CLAUDE.md`, `docs/PRD.md`, `docs/SSR.md` — 전반 규칙·스택·격리 경계.
- 현재 코드: `components/layout/SideMenu.tsx`, `SideMenu.test.tsx`, `ThemeToggle.tsx`, `AppHeader.tsx`, 기존 라우트 `app/settings`·`app/licenses`·`app/account`.

## 작업 방식

- UI 개선 작업이며 결정은 스펙·검증된 목업으로 확정돼 있다. 스펙 범위 안에서 추가 승인 없이 끝까지 자율 완성한다(TDD 준수).
- 시작 전, 내가 정하거나 해줄 것이 있으면 하나로 모아 물어보고(없으면 없다고 말하고 진행), 그 외엔 스펙 밖 에스컬레이션 사유가 생길 때만 멈춘다.

## 확정된 결정 (스펙 반영 — 재확인 불필요)

- 패널 구조(위→아래): 프로필 카드 → "메뉴" 그룹(설정·서비스소개·개발자정보·오픈소스라이선스) → "화면" 그룹(다크모드 토글) → "계정" 그룹(로그아웃) → 메타 푸터(버전·카피라이트).
- **아코디언 폐지**: 서비스 소개·개발자 정보를 제자리 아코디언이 아니라, 설정·라이선스와 같은 **이동 대상 화면**으로 만든다(네 항목 모두 chevron → 이동으로 통일).
- 프로필 카드: 아바타·이름·이메일·플랜 배지("얼리버드 무료 · Medium" 형태), 탭 시 /account. 멤버십 미구축 시 배지 우아한 폴백.
- 다크모드는 "화면" 그룹 인라인 토글(기존 ThemeToggle 재사용).
- 로그아웃은 "계정" 그룹(기존 세션 종료 로직).
- 푸터에 앱 버전(package.json version 주입, 하드코딩 금지) + "© 2026 getkkul · Made in Seoul".
- `RememberLogo` TODO 플레이스홀더는 실제 로고나 깔끔한 대체로 교체.
- 기존 접근성(포커스 트랩·ESC·오버레이·스와이프)·200ms 슬라이드·44px·라이트/다크는 **보존**(회귀 금지).

## 구현 참고

- 서비스 소개·개발자 정보를 별도 라우트로 만들지, 기존 컨벤션에 맞는 화면으로 만들지는 코드베이스 관례를 따르되, **네 항목이 반드시 동일한 이동 방식**이어야 한다(일관성이 요구사항).
- 사용자 정보는 기존 세션/프로필에서, 플랜은 subscriptions_membership에서(없으면 폴백).
- 인터랙티브 목업(getkkul-sidemenu-mockup.html)의 구조·동작을 시각 기준으로 참고한다(색·여백·그룹핑·이동/토글 구분).

## 품질 기준 (TDD)

- 각 AC를 실패 테스트로 먼저 작성한 뒤 구현한다. 기존 `SideMenu.test.tsx`도 재설계에 맞게 갱신한다.
- 특히 테스트 우선:
  - "메뉴 4항목이 모두 이동 패턴(아코디언 미사용)".
  - "프로필 카드 렌더 + 탭 시 /account 이동 + 플랜 배지 폴백".
  - "푸터 버전이 package.json 기반".
  - "로그아웃이 세션 종료 호출".
  - "포커스 트랩·ESC·오버레이·스와이프 닫기 보존".
- 버전은 빌드 시 주입(NEXT_PUBLIC_APP_VERSION 또는 생성 상수). 하드코딩 금지.

## 수용성 검증

- spec J절 V1~V7을 검증 대상으로 삼는다. 통과 기준: 전부 통과, 특히 V1(패널에 아코디언 0개)·V6(접근성 보존)은 필수.
- 자동 테스트로 가능한 부분(상호작용·라우팅·렌더·보존 동작)은 전부 테스트로 커버하고, 사람이 눈으로 확인할 체크리스트(패널 열기→각 항목 이동→다크토글→로그아웃→푸터 버전)를 함께 제시한다.

## 문서 정합성

- 사이드 메뉴 구조가 바뀌므로 관련 문서/주석을 갱신하고, 변경은 `docs/adr/`에 기록한다.
